## Описание проекта
Приложение для управления личными финансами.
Проект позволит пользователям добавлять доходы и расходы, устанавливать бюджеты на категории и получать статистику по своим финансам.

## Основные задачи:
Для успешного выполнения проекта ваше backend-приложение для управления личными финансами должно соответствовать следующим требованиям:
1. Авторизация пользователей
   - Реализовать функциональность для авторизации пользователей по логину и паролю. Приложение должно поддерживать несколько пользователей.

2. Функционал управления финансами
   - Разработать логику для добавления доходов и расходов. Пользователь должен иметь возможность создавать категории для планирования бюджета.
   - Предусмотреть функциональность для установления бюджета на каждую категорию расходов.
   
3. Работа с кошельком пользователя:
   - Привязать кошелёк к авторизованному пользователю. Кошелёк должен хранить информацию о текущих финансах и всех операциях (доходах и расходах).
   - Сохранять установленный бюджет по категориям.

4. Вывод информации:
   - Реализовать возможность отображения общей суммы доходов и расходов, а также данных по каждой категории.
   - Выводить информацию о текущем состоянии бюджета для каждой категории, а также оставшийся лимит.
   - Поддерживать вывод информации в терминал или в файл.

5. Подсчет расходов и доходов:
   - Разработать методы, подсчитывающие общие расходы и доходы, а также по категориям.
   - Поддержать возможность подсчета по нескольким выбранным категориям. Если категория не найдена, уведомлять пользователя.

6. Проверка вводимых данных:
   - Валидация пользовательского ввода и уведомление о некорректных данных.

7. Оповещения:
   - Оповещать пользователя, если превышен лимит бюджета по категории или расходы превысили доходы.

8. Сохранение данных:
   - При выходе из приложения сохранять данные кошелька пользователя в файл.
   - При авторизации загружать данные кошелька из файла.


## Структура проекта
### Папки
- src: папка с исходным кодом
- resources: папка с файлами в которых хранятся данные для работы программы
  - budgets.txt -- файл для хранения и установки бюджетов на категории расходов.
  - categories.txt -- файл для хранения категорий расходов и доходов.
  - transactions.txt -- файл для хранения транзакций (операции, доходы и расходы).
  - users.txt -- файл для хранения данных о разных пользователях
- output: папка в которую выгружаются отчеты по бюджетам

### Пакеты
#### Entities
Пакет предназначен для создания базовых сущностей в которых хранятся данные. 
Создания их структуры, структуры файлов в которых они будут храниться и для записи их в файлы.

**Классы**
- User: класс отражающий сущность пользователя
- Budget: класс отражающий сущность бюджета, который пользователь устанавливает для категории расходов
- Category: класс отражающий сущность категории доходов или расходов для конкретного пользователя
- CashFlow: enum-класс отражающий два типа категорий, доходы и расходы
- Transaction: класс отражающий сущность транзакции пользователя 
(под транзакцией подразумевается запись создаваемая пользователем, 
в которой он указывает категорию дохода или расхода на конкретную дату и сумму дохода или расхода)

**Интерфейсы**
- FileAccess: интерфейс отражающий возможность записи в файл (то есть сохранения базовой сущности в виде записи в какой-либо файл)
<br>Данный интерфейс имплементируют классы Budget, Category, Transaction и User.

#### Services
Пакет предназначен для создания "сервисов" по управлению базовыми сущностями. 
Представляет прослойку между слоем, с которым взаимодействует пользователем и слоем с базовыми сущностями и файлами.

- UsersManager: класс для управления пользователями
- CategoriesManager: класс для управления категориями
- BudgetManager: класс для управления бюджетами
- TransactionsManager: класс для управления транзакциями
- Statistics: класс для создания различных расчетов

### Прочие классы
- App: класс в котором отражена работа с пользователем и происходит обращение к сервисному уровню приложения на основе выбора пользователя
- Main: класс, точка входа в приложение, осуществляющий запуск программы

можно использовать тестовый аккаунт:
- Логин: test
- Пароль: test

Заведенные категории: 
  - Доход : Зарплата; 
  - Расход : Продукты; 
  - Расход : Образование; 
  - Расход : Одежда и обувь

Бюджеты установлены для следующих категорий: 
- Продукты 
- Одежда и обувь

### Описание работы программы
1. Авторизация пользователя
   - Войти в аккаунт
   - Зарегистрироваться (создать новый аккаунт)
   - Выйти

    При авторизации программа запоминает идентификатор (UUID) пользователя и все дальнейшие операции проводит с учетом полученного 
    идентификатора для фильтрации уже имеющихся записей и создания новых записей с текущим идентификатором пользователя.


2. После авторизации пользователя, программа предлагает выбрать один из вариантов дальнейшего действия:
   - **Просмотреть доходы-расходы**<br> 
    Происходит суммирование всех доходов и расходов. Вывод сумм по отдельности и вывод их разницы.
   
   - **Просмотреть бюджет для категории** <br>
    Пользователю необходимо ввести название категории по которой он хочет посмотреть текущее состояние бюджета. 
    На экран выведется остаток лимита для данного бюджета
   
   - **Сформировать отчет по всем бюджетам** <br>
     - Происходит получение всех бюджетов для пользователя
     - Идентификаторы категории заменяются на названия категорий 
     - Так же происходит расчет текущего остатка от выставленного лимита
     - Пользователю предложат выбор вывести отчет в консоль или сформировать файл 
     - При выборе вывода в файл, программа предлагает ввести имя файла (формат по умолчанию .txt)
   
   - **Создать транзакцию** <br>
     - Пользователю необходимо ввести тип категории (Доход / Расход), категорию, сумму и дату транзакции.
     - При создании новой транзакции предусмотрены следующие проверки:
       - Проверка общего баланса (не превысили ли доходы расходы)
       - Проверка на превышение лимита бюджета для категории по которой создается транзакция
   
   - **Создать бюджет** <br>
    Пользователю необходимо ввести название категории для которой он хочет установить бюджет (тип категории по умолчанию устанавливается "Расход"),
    лимит трат по данной категории, текущие траты (если они есть), дату начала и дату конца действия лимита.
   
   - **Создать категорию** <br>
    Пользователю необходимо ввести тип категории (Доход / Расход) и название категории.
   
   - **Выйти** <br>
    Завершение работы с программой

> Так как в ТЗ к проекту не требовалась реализации работы с датами, сейчас нет подсчета доходов-расходов по дням, месяцам, годам.
><br> Не учитываются начальная и конечная даты бюджетов.
><br> Но в модель хранения данных заложены поля для реализации работы с датами на будущее

Например:
1. Действие А
2. Действие Б
